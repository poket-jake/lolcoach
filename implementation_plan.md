# Vercel 배포 구현 계획

이 계획은 정적 HTML/JS/CSS로 구성된 '브실골플' 프로젝트를 Vercel 플랫폼에 배포하는 절차를 담고 있습니다.

## 1. 프로젝트 설정 보강
Vercel은 `package.json`이 없어도 정적 파일을 자동으로 인식하지만, 프로젝트 관리와 명확한 의도를 위해 설정을 추가합니다.

### 1.1 `package.json` 생성
- 프로젝트 이름, 버전, 그리고 로컬 실행을 위한 스크립트를 포함합니다.
- `serve` 패키지를 사용하여 로컬에서 배포 전 최종 확인을 할 수 있도록 합니다.

### 1.2 `vercel.json` 생성 (선택 사항)
- 캐싱 전략이나 클린 URL 설정을 위해 기본 설정을 작성합니다.

## 2. 배포 가이드라인 작성
사용자가 직접 배포를 완료할 수 있도록 두 가지 경로를 제시합니다.

### 2.1 GitHub 연동 배포 (권장)
- 가장 표준적인 방법이며, 코드 수정 시 자동으로 재배포됩니다.
1. GitHub 저장소 생성 및 코드 Push.
2. Vercel 사이트 접속 후 'Add New Project'.
3. GitHub 저장소 Import 및 배포.

### 2.2 Vercel CLI 배포
- 터미널에서 즉시 배포할 수 있는 방법입니다.
1. `npm i -g vercel` 설치.
2. `vercel` 명령어로 로그인 및 프로젝트 연결.
3. 배포 승인.

## 3. 검증
- 배포된 URL이 정상적으로 작동하는지 확인합니다.
- 이미지(logo.png) 및 외부 API(DDragon) 데이터가 올바르게 렌더링되는지 체크합니다.
